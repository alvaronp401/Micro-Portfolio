# 🌿 Skill: Git Workflow Profissional

No mercado, o seu histórico de commits conta a história do seu trabalho.

## 🏗️ Padrões de Commit (Conventional Commits)
1. **feat**: Uma nova funcionalidade.
2. **fix**: Correção de um erro.
3. **refactor**: Mudança no código que não altera funcionalidade.
4. **docs**: Mudanças na documentação.

## 🚀 Exemplo Senior
```bash
git commit -m "feat(server): implementar endpoint de listagem de projetos"
git commit -m "fix(client): ajustar alinhamento do botão de contato"
```

## 📝 Para o Notion
**Git - Como fazer commits profissionais** -> Use prefixos como `feat:`, `fix:` ou `refactor:` seguido de uma descrição curta e clara no presente.
